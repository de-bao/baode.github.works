# 后台管理页面使用指南

## 📍 访问地址

后台管理页面已创建在 `admin/index.html`

### 部署方式（推荐）

#### 方式 1: 部署到 Cloudflare Pages（推荐）

1. 在 Cloudflare Dashboard 创建新的 Pages 项目
2. 连接 GitHub 仓库（如果已上传）或直接上传 `admin` 文件夹
3. 构建命令留空，输出目录设置为 `admin`
4. 部署后访问：`https://你的pages域名/admin/`

#### 方式 2: 部署到 GitHub Pages

1. 将 `admin/index.html` 复制到你的 Jekyll 项目的 `admin/` 目录
2. 推送到 GitHub，GitHub Pages 会自动部署
3. 访问：`https://de-bao.github.io/admin/`

#### 方式 3: 本地使用

直接在浏览器中打开 `admin/index.html` 文件即可使用。

## ✨ 功能说明

### 1. 文章管理
- ✅ 查看所有文章（全部/已发布/草稿）
- ✅ 创建新文章
- ✅ 编辑文章
- ✅ 删除文章
- ✅ 发布文章（生成 Jekyll Markdown 格式）

### 2. 文章编辑
- 标题（必填）
- 摘要
- 内容（Markdown 格式）
- 标签（逗号分隔）
- 分类（逗号分隔）
- 日期
- 阅读时间（分钟）

### 3. 发布功能
点击"发布文章"后，系统会：
1. 将文章状态改为 `published`
2. 生成 Jekyll 格式的 Markdown 文件内容
3. 返回完整的 Front Matter 和内容

你可以将返回的内容保存到 `_posts/` 目录，文件名格式：`YYYY-MM-DD-slug.md`

后台页面默认使用以下 API 地址：

```javascript
const API_BASE = 'https://api.debao.qzz.io/api';
```

如果需要修改，编辑 `admin/index.html` 中的 `API_BASE` 变量。

## 📝 使用流程

1. **登录**
   - 打开后台页面
   - 输入用户名和密码登录

2. **创建文章**
   - 点击"新建文章"
   - 填写文章信息
   - 点击"保存草稿"或"发布文章"

3. **编辑文章**
   - 在文章列表中点击"编辑"
   - 修改内容后保存

4. **发布文章**
   - 编辑文章后点击"发布文章"
   - 或直接在列表页点击"发布"按钮
   - 系统会生成 Jekyll Markdown 格式的内容

5. **同步到 Jekyll**
   - 发布后，可以通过 API 获取 Markdown 内容
   - 或使用 GitHub API 自动提交到仓库

## ⚠️ 注意事项

1. **认证系统**：当前是简化版，生产环境建议改进
2. **Token 存储**：使用 localStorage，关闭浏览器后需要重新登录
3. **CORS**：已配置允许所有来源，生产环境建议限制域名
4. **文件上传**：需要先启用 R2 存储功能
